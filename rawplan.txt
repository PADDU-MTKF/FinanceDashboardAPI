Finance dashboard system API
--------------------------------

User and Role managment
User(username, name, role,status,hashedpasswrod)
GodUser(One user that no one can remove) > master admin(manage users and/or all other feature) > Admin(Full access to CRUD and insights) > Analyst(Only view all data and insights) > Viewer(Only view, no insights)
    Only master Admin can access the below function
        createUsers() -> create one or multiple useres
        updateRole() ->  Assign/update the role of a user
        updateUserStatus() -> change user from active or inactive
        listAllUsers()


Financial Records Management

transactions(amount,type[income/expense],category,notes)

Only admins
    addTransaction() -> add new data by authorised users 
    updateTransaction() -> update the fields of the data
    deleteTransaction()
All Users
    getTransaction(page=0,filter=None) -> default return all trans from the specified page, else filter 



Dashboard Summary APIs

Viewers cant access this features
    getInsights(totalIncome,totalExpense,netBalence,categoryTotal,recentAct,topCategory)



Who can access what?

Viewer -> getTransaction()
Analyst -> getTransaction(), getInsights()
transactionAdmin -> getTransaction(), getInsights(), addTransaction(), updateTransaction(), deleteTransaction()
userAdmin -> createUsers(), updateRole(), updateUserStatus(), listAllUsers()
masterAdmin -> hybrid of transactionAdmin and userAdmin
